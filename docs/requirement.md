### 项目名称：Hermes-OpenClaw 多智能体协同可视化监控台

#### 1. 文档版本与背景

| 字段 | 内容 |
| :--- | :--- |
| **版本号** | V1.0 |
| **撰写日期** | 2026-04-20 |
| **前置条件** | 已完成 **编排层** 开发，Hermes 与 OpenClaw 间可通过 API 进行任务分解与分发。 |

#### 2. 项目目标
本需求旨在构建一个 **Web 可视化操作界面**，用于替代现有的命令行或 API 调试工具。该界面需作为用户与底层多智能体系统交互的唯一入口，实现对复杂任务从输入、分解、分发到执行反馈的**全链路透视**。

#### 3. 核心用户场景
- **场景一：任务发起**：用户在输入框中键入“分析竞争对手 X 的最新动态并生成摘要”，点击提交后，界面需立即在画布上绘制出 Hermes 主控节点。
- **场景二：过程监控**：用户能够看到 Hermes 将任务拆解为“网络搜索员”、“文档分析师”两个子任务，并看到这两个任务分别流向 OpenClaw 子 Agent 的动画过程。
- **场景三：内容审计**：当某个子 Agent 执行出错或返回幻觉内容时，用户可通过点击该 Agent 节点，在右侧抽屉中查看其完整的上下文对话记录。

#### 4. 功能性需求

##### 4.1 任务输入与提交模块
| 需求编号 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| **F-01** | 提供 **Markdown 编辑器风格的输入框**，支持多行文本输入及简单的格式快捷键。 | P0 |
| **F-02** | 提供 **模型参数快速配置栏**，允许在提交前临时调整 Hermes 的温度值或指定使用的模型版本。 | P1 |
| **F-03** | 点击 **“启动任务”** 按钮后，界面进入“监听状态”，按钮转变为不可用的“执行中...”。 | P0 |

##### 4.2 拓扑可视化与交互流展示
| 需求编号 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| **F-04** | **动态节点渲染**：基于编排层返回的任务 DAG，自动绘制节点图。节点需通过颜色区分角色：<br> - 🔵 **Hermes 节点**：任务分解与汇总。<br> - 🟢 **OpenClaw 子节点**：具体执行单元。<br> - 🟡 **工具节点**：API 调用或数据库查询。 | P0 |
| **F-05** | **实时连线动画**：当数据包从 Hermes 流向子 Agent 时，连线需呈现 **流动的光点特效**；若子 Agent 处于“思考中”状态，对应节点需有呼吸灯效果。 | P0 |
| **F-06** | **状态标识同步**：节点需准确反映编排层状态：`PENDING`、`RUNNING`、`SUCCESS`、`FAILED`。失败节点需高亮红框并支持悬浮查看错误堆栈。 | P0 |

##### 4.3 对话内容与上下文实时面板
| 需求编号 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| **F-07** | **右侧动态抽屉**：点击画布上任意一个 Agent 节点，界面右侧滑出 **实时对话日志面板**。 | P0 |
| **F-08** | **流式渲染内容**：对话面板需支持 **SSE 或 WebSocket 流式输出**，像 ChatGPT 一样逐字显示子 Agent 的回复内容，而非等待全部完成后一次性刷出。 | P0 |
| **F-09** | **上下文追溯**：面板内需展示该子 Agent 接收到的 **System Prompt** 片段及具体的子任务描述，以便调试幻觉问题。 | P1 |

##### 4.4 历史任务回放
| 需求编号 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| **F-10** | 左侧栏需列出 **近期会话列表**，点击后可进入 **只读模式**，重现当时编排层的拓扑结构和对话时间轴。 | P1 |

#### 5. 非功能性需求

| 需求编号 | 类别 | 详细描述 |
| :--- | :--- | :--- |
| **NF-01** | **实时性** | 界面与编排层 WebSocket 连接的延迟需控制在 **< 100ms**，确保动画与后端日志同步。 |
| **NF-02** | **兼容性** | 需完美适配 **Chrome 120+** 及 Edge 浏览器，支持 **深色模式** 切换。 |
| **NF-03** | **错误兜底** | 若编排层服务中断，界面需在 3 秒内弹出全局提示并尝试自动重连，**禁止界面白屏卡死**。 |

#### 6. 界面布局参考建议

| 区域 | 占比 | 组件描述 |
| :--- | :--- | :--- |
| **顶部栏** | 8% | 显示系统标题、当前会话 ID、连接状态指示灯。 |
| **左侧边栏** | 15% | 历史任务列表、当前活跃子 Agent 数量统计。 |
| **中央画布区** | 55% | 基于 **React Flow** 或 **G6** 绘制的 Agent 交互拓扑图。 |
| **右侧详情区** | 22% | 折叠/展开的对话流面板，包含 Markdown 渲染引擎。 |
| **底部输入区** | 固定高度 | 悬浮式输入框，不影响画布操作。 |

#### 7. 验收标准
1.  输入任务 **“帮我查一下北京和上海的天气，对比后推荐穿衣”**，画布上必须出现至少 **2 个 OpenClaw 子 Agent 节点**。
2.  当右侧面板显示“正在调用天气 API...”时，对应节点需处于 **RUNNING 呼吸状态**。
3.  任意子 Agent 若返回 `500` 错误，画布对应节点需立即变红并停止后续任务分发。
